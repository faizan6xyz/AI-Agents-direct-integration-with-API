Setup steps: 

    1. Meta Developer + Business Manager

    Create a Meta Developer account at developers.facebook.com.
    Create/verify a Meta Business Manager account — you'll need business info and go through SMS/voice verification.

    2. Create the app

    Create a new Business-type app on the developer dashboard.
    Select the "Connect with customers through WhatsApp" use case → add the WhatsApp product.
    Connect it to a WhatsApp Business Account (WABA) — create new or select existing.

    3. Get a test number (free, instant)

    Meta gives you a free test phone number (sandbox) plus your WABA ID and access to the WhatsApp Manager panel in Meta Business Suite.
    You can send test messages to a few verified numbers immediately — no waiting.

    4. Access tokens

    Two types: a temporary token (24 hrs, from the dashboard, fine for early testing) and a permanent System User token (for production — generate via Business Manager → System Users).

    5. Send your first message

    From the API Setup page, use the temp token + test number to fire off a curl call to graph.facebook.com/v21.0/{phone-number-id}/messages. Meta's Quickstart walks this exact call.

    6. Set up a webhook

    Required to receive incoming messages / delivery statuses. You need a public HTTPS endpoint (a FastAPI/Flask route works fine, or ngrok while testing) that verifies Meta's challenge token and then receives POST events.

    7. Move to a real number + business verification

    Register your actual business phone number (must be clean — not already tied to a personal WhatsApp account).
    Complete business verification (tax ID, incorporation docs, utility bill) — takes 2-10 business days. Without it, you're capped at 250 conversations/24h.

    8. Message templates

    Any message you send outside a 24-hour customer-initiated window must be a pre-approved template (submit via WhatsApp Manager, approval usually within hours-days).
