Setup :
    1. Create a Google Cloud project
        Go to console.cloud.google.com, create a new project (or reuse an existing one), and note the project name.

    2. Enable the Drive API
        In the Cloud Console: APIs & Services → Library → search "Google Drive API" → click Enable.

    3. Configure the OAuth consent screen
        APIs & Services → OAuth consent screen (now under "Google Auth platform"):
            Choose User type: "Internal" if you're on Google Workspace and only your org will use it, otherwise "External"
            Fill in app name, support email
            Under scopes, you can skip for now — you'll request them in code
            Accept the Google API Services User Data Policy

    4. Create OAuth credentials
        Google Auth platform → Clients → Create Client:
            Application type: Desktop app (simplest for scripts/local tools), or Web application if this runs server-side with a redirect URI
            Download the resulting JSON, save it as credentials.json in your project directory

Scopes :

    Non-sensitive (basic verification only)
        https://www.googleapis.com/auth/drive.appdata (aka drive.appfolder) — view/manage the app's own configuration data
        https://www.googleapis.com/auth/drive.install — lets your app appear in Drive's "Open with" / "New" menu
        https://www.googleapis.com/auth/drive.file — create new files, or modify only files the user opened with or explicitly shared to your app (e.g. via Google Picker)

    Sensitive
        https://www.googleapis.com/auth/drive.apps.readonly — view apps authorized to access the user's Drive

    Restricted (require restricted-scope verification + security assessment if you store/transmit the data)
        https://www.googleapis.com/auth/drive — full read/write access to all Drive files
        https://www.googleapis.com/auth/drive.readonly — view and download all Drive files
        https://www.googleapis.com/auth/drive.activity — view and add to the activity record of files
        https://www.googleapis.com/auth/drive.activity.readonly — view the activity record only
        https://www.googleapis.com/auth/drive.meet.readonly — view Drive files created/edited by Google Meet
        https://www.googleapis.com/auth/drive.metadata — view and manage file metadata
        https://www.googleapis.com/auth/drive.metadata.readonly — view metadata only
        https://www.googleapis.com/auth/drive.scripts — modify your own Apps Script scripts' behavior
