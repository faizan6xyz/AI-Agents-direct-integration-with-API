Setup requirements

    1. PayPal Developer account — sign up free at https://developer.paypal.com
    
    2. Create an App in the Developer Dashboard → gives you a Client ID and Secret (sandbox and live each have their own).
    
    3. Sandbox test accounts — PayPal auto-generates sandbox buyer/seller accounts under "Sandbox → Accounts" so you can test payments without real money.
    
    4. Python package: pip install requests
    
    5.Environment:
        Use https://api-m.sandbox.paypal.com while testing.
        Switch to https://api-m.paypal.com once you go live (with your live Client ID/Secret).

    6. Return/Cancel URLs — in production these should be real endpoints on your website/server (e.g. a /payment-success route) so PayPal can redirect the client back after they approve payment.

    7.(Recommended for production) Set up a webhook in the Developer Dashboard for the CHECKOUT.ORDER.APPROVED / PAYMENT.CAPTURE.COMPLETED events instead of manually polling — that way your server gets notified automatically instead of you having to re-check status yourself.
